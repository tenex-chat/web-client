Agent Profile Metadata Conversion

This feature enables TENEX to parse agent-specific metadata from Nostr kind:0 (profile) events and convert them into structured NDKAgentDefinition events for better compatibility and features.

Key Features Implemented

  • Metadata Parsing: Extract agent-specific fields from kind:0 events (system prompt, instructions, use criteria)
  • Visual Indicators: Show when an agent has metadata but no definition
  • Conversion UI: One-click conversion button with pre-populated form
  • Data Migration: Seamless transfer of all agent metadata to the new format

User Experience

When viewing an agent profile that only has kind:0 metadata:

  1. The profile page displays the agent's metadata from their kind:0 event
  2. A prominent "Convert to Agent Definition" button appears with a sparkle icon
  3. An informational card explains the benefits of conversion
  4. Clicking the button opens a pre-populated form with all existing metadata
  5. User can review, edit, and add additional configuration
  6. Upon submission, a new NDKAgentDefinition event is published

Testing the Feature

To test this feature:
  1. Navigate to an agent profile that only has kind:0 metadata (no kind:4199 event)
  2. Look for the "Convert to Agent Definition" button in the header
  3. Check the Details tab for the conversion prompt card
  4. Click the conversion button to open the pre-filled dialog
  5. Review the imported data and complete the conversion
